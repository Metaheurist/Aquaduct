# Desktop UI (PyQt6)

## Layout
- Package: [`UI/`](../../UI/) (theme, workers, main window, `tabs/` per screen)
- **Mouse wheel**: dropdowns, numeric spins, and **quantization sliders** do **not** change value when the pointer is over them—scrolling moves the **page or scroll area** instead. See [`UI/widgets/no_wheel_controls.py`](../../UI/widgets/no_wheel_controls.py) (`NoWheelComboBox`, `NoWheelSpinBox`, `NoWheelDoubleSpinBox`, `NoWheelSlider`).
- **Title bar** (frameless window): drag by the title row; **Photo \| Video** ([`UI/media_mode_toggle.py`](../../UI/media_mode_toggle.py)) switches **`media_mode`** (`video` \| `photo` in `ui_settings.json`) — output root **`.Aquaduct_data/videos`** vs **`.Aquaduct_data/pictures`**, tab set (**Video** vs **Picture**, Captions/Effects only in video mode), and Library rescan targets; **Save** (floppy icon), **resource graph** (chart icon), **Help** (?), and **Close** (✕) use vector icons drawn by [`UI/widgets/title_bar_svg_icons.py`](../../UI/widgets/title_bar_svg_icons.py) inside [`UI/title_bar_outline_button.py`](../../UI/title_bar_outline_button.py) (**`TitleBarOutlineButton`**, `icon_kind=…`) so outlines stay **antialiased** and icons stay sharp on HiDPI. **📈** opens a live **resource usage** graph (this process CPU / RAM / **CUDA VRAM** when CUDA is on — the **Monitor** dropdown lists each CUDA device plus **Split view — all GPUs**; choices persist as `resource_graph_monitor_gpu_index` and `resource_graph_split_view` in `ui_settings.json`; see [Config](../reference/config.md)), 1s updates; tooltip explains **Auto** GPU routing vs per-GPU charts). During long **`from_pretrained`** loads, **`load_heartbeat`** lines log to stderr and mirror the latest line into the graph window **footer**. During a run, **`PipelineWorker`** may emit optional **VRAM / pipeline notices** (non-blocking) when [`pipeline_notice`](../../src/runtime/pipeline_notice.py) is active — for example low free VRAM before a heavy load ([`vram_watchdog`](../../src/util/vram_watchdog.py)). **Help** opens the tutorial (topics on the left, slide-style pages with **Previous** / **Next**); **Close** exits the app.
- **First-run tutorial**: on the first launch after install (when `tutorial_completed` is false in `ui_settings.json`), the same Help window opens automatically after startup (delayed so it does not stack on the optional Hugging Face token dialog). Dismissal saves `tutorial_completed: true`. Use **?** anytime to reopen help.
- **Help links in hints**: several controls use rich tooltips with **Open in Help →** ([`UI/tutorial_links.py`](../../UI/tutorial_links.py)). Hover shows a small popup (not the native tooltip) so the link is clickable; it opens Help on the matching **topic** and **slide**. Title bar Save / graph / Help and various Run / Topics / Video / Tasks / Library / Model / Characters hints include this when they map cleanly to tutorial content.
- **Section composition**: Prefer grouping settings with **`section_card()`** + **`section_title()`** from [`UI/tab_sections.py`](../../UI/tab_sections.py) (styled as `QFrame#SettingsSectionCard` in [`UI/theme.py`](../../UI/theme.py)). Keep one short subtitle per block; move long explanations to tooltips or Help so tabs stay scannable.
- Launcher script: [`UI/ui_app.py`](../../UI/ui_app.py) (same as `python -m UI`)

## Purpose
Provide a desktop control panel for **Aquaduct** with a **TikTok-style dark theme** and tabbed settings.

## Frozen Windows EXE

Build, smoke test, and `-debug` behavior: [building_windows_exe.md](../build/building_windows_exe.md) and [`../build/README.md`](../../build/README.md). Import-time notes: [performance.md](../pipeline/performance.md).

## How to run

```powershell
python -m UI
```

Or:

```powershell
python UI/ui_app.py
```

On cold start (especially a **frozen Windows EXE**), the app shows a small **startup splash** with a **progress bar** and **elapsed seconds** while settings load and the main window module imports (the long stretch uses an **indeterminate** bar because imports run on one thread). Set **`AQUADUCT_NO_SPLASH=1`** to skip the splash for debugging.

## Tabs
- **Run**
  - Sections (**Output**, **Script & content**, **Actions**) use **`section_card()`** + shared spacing and subsection titles ([`UI/tab_sections.py`](../../UI/tab_sections.py)). Pipeline status while a job runs is summarized on the **Tasks** tab; the **Run** button tooltip points there (and Help where linked).
  - **Run** button: starts the pipeline. **Preflight** runs first: in **Local** mode, required Hugging Face snapshots must exist under your **Model files location** (Script / Image / Voice / Video for video mode; Script / Image only in **Photo** mode); in **API** mode, Generation APIs must be configured with keys. If something is missing, the app switches to the **Model** tab and shows a dialog instead of starting. **Videos to generate** (`N > 1`) enqueues **N independent full runs** (each produces one video and one finished Tasks row); the first starts immediately and the rest wait in the FIFO queue. If a pipeline is **already running**, another click **queues** more runs (FIFO) — the same preflight runs before each queued job. Each queued job stores **settings** for one video (or preview/storyboard payloads) from when you clicked. **Stop** cancels the current job and **drops** the queue. While a run is active, **Run** stays enabled so you can enqueue more jobs.
  - **Content source**: **Preset (news cache + topics)** uses the deduped headline cache and per-format topic tags (same as before). **Custom (your instructions)** uses a multiline text box: the script LLM **expands** your notes into a creative brief, then **writes** the structured `VideoPackage` (two LLM passes — slower than Preset). Custom mode does **not** pick headlines from the cache; topic tags still bias hashtags when relevant. Instructions are stored in settings (`run_content_mode`, `custom_video_instructions`; length capped). Run / Preview / Storyboard preview require non-empty instructions when Custom is selected.
  - **Video format** (News / Cartoon / Explainer / **Cartoon (unhinged)** / Creepypasta / **Health advice**): together with Topics, selects which tag list the pipeline uses (`video_format` + `topic_tags_by_mode`). **News** and **Explainer** share the same AI/product headline sourcing and news-style script shape. **Cartoon**, **Unhinged**, and **Creepypasta** use Firecrawl-first creative or horror-fiction web seeds (not Google News headlines) for Preset runs and **Discover** where applicable. **Health advice** uses Firecrawl-first **wellness / health-education** web searches, clinician-shaped scripts, and the same topic-research pack path as creative modes under **`data/topic_research/health_advice/`**. **Cartoon (unhinged)** favors chaotic comedy/absurdist animation; it rotates **local** pyttsx3 voices per script beat, and if a character uses **ElevenLabs** (or a custom non-default voice), the app uses one voice for the whole track.
  - **Personality** (preset or Auto) and optional **Character** (see Characters tab) for script + storyboard consistency
  - Live pipeline / preview / storyboard progress is shown as the **top row** on the **Tasks** tab; the **Status** column shows **stage + percent** (e.g. script, voice, images, encode) for runs, not only “Running…”
  - Open `videos/` folder
  - Save settings
- **Topics**
  - Tag list and controls sit in a **section card**; keep the short intro line — longer behavior notes are on the **Mode** combo tooltip / Help.
  - **Mode** combo matches video formats; each mode has its own tag list (stored in `ui_settings.json` under `topic_tags_by_mode`)
  - Add/remove tags for the **selected mode** only; the tag line includes an optional **🧠** control to expand/improve text with the local **Script (LLM)** model ([`UI/brain_expand.py`](../../UI/brain_expand.py))
  - **Discover**: **News** and **Explainer** — headline-style suggestions from your tags. **Cartoon**, **Unhinged**, **Creepypasta**, and **Health advice** — Firecrawl searches the open web (enable Firecrawl on the **API** tab): memes/stories/horror fiction for creative modes; **wellness and health-education** pages for **Health advice**. Suggested lines come from page titles (including full-title fallback when parsing would otherwise be empty). Approved lines are added to that mode’s tag list. Each Discover run for modes that save a research pack also writes **`data/topic_research/<cartoon|unhinged|creepypasta|health_advice>/manifest.json`** and **`images/`** (preview / og:image downloads when available); the log shows that folder after a successful add. When you later run the pipeline with **Gather web context** and/or **Download reference images** (Video tab), the latest manifest is prepended into the same Firecrawl script-context path so the script LLM sees those links and local image paths.
- **Tasks**
  - The **Tasks** tab label shows **`Tasks (n)`** while work is active or queued: **n** counts the pipeline/preview/storyboard worker (if running), **queued** runs waiting behind the current job, and **TikTok** / **YouTube** upload workers started from the table. **Queued** pipeline jobs are listed as **separate rows** (“Queued pipeline run”, status **Waiting in queue…**) below the in-progress row so the badge matches what you see. Select a queued row and **Remove** to drop that waiting job instantly (nothing is written to disk yet); use **Stop** for the row that is actually running.
  - **Run controls** (refresh list, pause between steps, stop) sit **above** the task table, directly under the short intro text — icons are **SVG** ([`UI/widgets/toolbar_svg_icons.py`](../../UI/widgets/toolbar_svg_icons.py)); **Selected task** actions (**Open folder**, **Play video**, uploads, etc.) stay **below** the table.
  - Lists successful renders (`data/upload_tasks.json`): open folder, play `final.mp4`, copy caption from `meta.json` / `hashtags.txt`, mark posted manually, **Upload to TikTok** (inbox) and/or **Upload to YouTube** when the **API** tab is configured (separate toggles); optional auto-uploads per platform after each render
  - While a **pipeline run**, **Preview**, or **Storyboard preview** is active (top row), **Pause** / **Resume** waits between major steps ([`src/runtime/pipeline_control.py`](../../src/runtime/pipeline_control.py)); **Stop** cancels at the next checkpoint (does not interrupt mid–GPU generation). For an active **pipeline**, **Stop** also **clears** any extra runs you queued with **Run** while the previous job was still going.
- **Library**
  - Finished outputs (**`videos/`** or **`pictures/`**) and **`runs/`** workspaces are each in their own **section card** (titles follow **Photo** \| **Video** mode).
  - In **Video** mode, lists **`videos/`** projects that contain **`final.mp4`** (title from **`meta.json`**, folder name, modified time, output size). In **Photo** mode, lists **`pictures/`** projects (finished image outputs). Always lists **`runs/`** workspaces (intermediate files; **assets/** column when present). **Refresh** rescans; opening the tab also refreshes; a completed pipeline run refreshes the lists.
  - **Open … folder** / **Open runs folder** open **`.Aquaduct_data/videos`**, **`.Aquaduct_data/pictures`**, or **`runs`** depending on mode and toolbar choice. For a selected project row: **Open folder**, **Open assets** when applicable, **Play final.mp4** (video); double-click opens the project folder. For **runs/**: **Open run folder** and **Open assets** when **`assets/`** exists ([`UI/tabs/library_tab.py`](../../UI/tabs/library_tab.py), [`UI/library_fs.py`](../../UI/library_fs.py)).
- **Video** (tab visible in **Video** mode)
  - Major blocks (**Platform template**, **Output & timing**, **Quality / performance**, **Story pipeline**, **Advanced**) are separated with vertical spacing instead of horizontal rules between sections ([`UI/tab_sections.py`](../../UI/tab_sections.py)).
  - **Platform template** (top): **tiles** (like graphics presets in a game) — one card per social profile (e.g. short-form vertical, Instagram square/portrait, Pinterest, YouTube/LinkedIn landscape) plus **Pro — vertical 60 fps** plus **Custom**. Clicking a tile applies resolution, FPS, micro-scene timing, images per video, bitrate, motion-mode scene timings (`clips_per_video` / `clip_seconds`), and pro fields when applicable. Changing any of those controls switches to **Custom**. Stored as `video.platform_preset_id` in settings; definitions in [`src/settings/video_platform_presets.py`](../../src/settings/video_platform_presets.py).
  - **Resolution** (dropdown of width×height presets used by the templates)
  - Images per video (hidden when **Pro mode** is on)
  - **Pro mode** + **Pro scene length (seconds)**: in the desktop UI, Pro **turns slideshow off** and runs **multi-scene video** from the script: **text-to-video** models (e.g. ZeroScope) render segments directly; **image-to-video** models (e.g. Stable Video Diffusion) first generate **keyframes** with the **Image** model, then animate them. Narration is aligned to the combined segment length. If you force **slideshow + Pro** via edited settings, the older **one still per output frame** path still exists (round(length × FPS) frames; preflight warns on large counts).
  - Slideshow mode vs **motion mode** (generate images and stitch vs **scene**-based video pipeline when slideshow is off)
  - FPS
  - Micro-scene min/max seconds (slideshow timing)
  - Bitrate preset (low/med/high)
  - Export intermediate micro-scenes toggle
  - Delete generated images after run (save storage)
  - **Allow NSFW image output** (optional): disables the diffusion safety checker for image generation so flagged frames are not replaced with black slides — use only where appropriate; you are responsible for platform rules (`AppSettings.allow_nsfw`)
- Quality/performance and advanced controls live under **Video**:
  - Prefer GPU toggle (advisory)
  - **Resume partial pipeline** (persisted via `resume_partial_pipeline` in **`ui_settings.json`**): coarse **`assets/run_checkpoint.json`** + **`pipeline_script_package.json`** in each video folder; incomplete projects may trigger a **startup resume** prompt (**Yes / No / Discard checkpoint via Help**) when fingerprint matches configured script/image/video/voice ids. Outputs can pin to the resumed folder (**`resume_partial_project_directory`** is session-only — not saved to disk).
  - High quality topic selection, fetch article text, prompt conditioning (no model-specific toggles here; script/image behavior is chosen on the **Model** tab)
  - **Story pipeline (LLM)** (optional): **Multi-stage script review** runs extra local LLM passes after the first draft (beat structure, safety, length, clarity for News/Explainer/**Health advice**; dialogue, pacing, punchlines for Cartoon/Unhinged; horror arc for Creepypasta). **Gather web context** uses **Firecrawl** (API tab) to search/scrape a short digest saved under the run folder and fed into script generation. For **Cartoon** and **Unhinged**, searches bias toward **memes / viral / templates** with extra Firecrawl queries so the digest includes meme-adjacent pages (not generic news). For **Health advice**, supplemental queries bias toward **wellness / nutrition / mental health** pages. **Download reference images** pulls images from scraped pages (a few more in “rich web” modes: cartoon, unhinged, creepypasta, health_advice) and uses the first as an **img2img** init for the first generated frame when the image model supports it (SDXL-style paths). For modes that persist **Topics → Discover** research, the latest pack under **`data/topic_research/<mode>/`** is merged into that same `extra_markdown` path when context gathering runs, so script generation can use your saved Discover pages and reference image filenames. These options add latency and API usage when Firecrawl is involved.
  - Background music picker
  - Clear news URL/title cache: removes legacy `seen.json` / `seen_titles.json` and all `seen_*.json` / `seen_titles_*.json` under `data/news_cache/`
- **Picture** (tab visible in **Photo** mode)
  - Templates, dimensions, output type (single image, pack, layouted sheet), **picture format** presets aligned with Run, and photo-specific branding options. Pipeline writes under **`.Aquaduct_data/pictures/<project>/`** ([`media_output_root()`](../../src/core/config.py), [`UI/tabs/picture_tab.py`](../../UI/tabs/picture_tab.py)).
- **Captions** (video mode; tab hidden in **Photo** mode)
  - Word-by-word captions and optional **Key facts** card. The facts card is **drawn only** when **Video format** (Run tab) is **News**, **Explainer**, or **Health advice**; **Cartoon**, **Cartoon (unhinged)**, and **Creepypasta** runs skip it regardless of the Captions toggle ([`video_format_supports_facts_card`](../../src/core/config.py)).
- **Branding**
  - Optional full-theme palette overrides (presets or custom hex + color picker). Changing the **Palette** dropdown updates the hex swatches and fields for that preset (Custom unlocks per-row overrides). Named presets include **Default (TikTok dark)**, **Ocean**, **Sunset**, **Monochrome**, **Amber night**, **Dracula**, **Ember**, **Forest**, **Lavender**, **Nord night**, **Rose**, **Slate**, and **Custom** — see [`UI/theme/palette.py`](../../UI/theme/palette.py) (`PRESET_PALETTES`). Copy is kept short; optional details in tooltips.

- **Characters**
  - Create, edit, and delete **characters** (name, identity, visual style, negative prompts, voice overrides). Stored in `data/characters.json` (local; not committed). The tab uses **section cards** to separate the character list / preset row from the profile editor. The list toolbar (**add** / **duplicate** / **delete**) uses **SVG** icons ([`UI/widgets/toolbar_svg_icons.py`](../../UI/widgets/toolbar_svg_icons.py)). **Preset** + **Generate with LLM** fills fields from built-in archetypes using the **Model** tab script model ([Characters](characters.md), [brain](../pipeline/brain.md)).
  - Multi-line fields can use the **🧠** control to expand/improve text with the **Script (LLM)** model — same repo id as the **Model** tab combo ([`resolve_llm_model_id`](../../UI/brain_expand.py), [Models](../reference/models.md)).

- **API**
  - **Generation APIs** (first block): per-role **Provider** and **Model** (keys: saved OpenAI/Replicate token where applicable, plus **env** overrides — `GEMINI_API_KEY`, `SILICONFLOW_API_KEY`, `KLING_ACCESS_KEY` + `KLING_SECRET_KEY`, `MAGIC_HOUR_API_KEY`, `INWORLD_API_KEY`, etc.). Suggested first-row entries include **Google AI Studio (Gemini)** (script), **SiliconFlow** (stills), **Kling AI** (Pro *motion*), **Magic Hour** or **Replicate** (alternates), and **Inworld** or **OpenAI** TTS; optional **ElevenLabs** (below) for cloud TTS. The block is shared with the **Model** tab when that tab is in **API** mode (one panel is reparented). See [API generation](../integrations/api_generation.md). In **Photo** mode, only **LLM** and **Image** API roles are required for a run (no voice or cloud video clips).
  - Hugging Face, Firecrawl, ElevenLabs, TikTok, and YouTube blocks are grouped in **section cards** with a short top-level subtitle (env overrides described in tooltips where relevant).
  - Hugging Face token (optional; helps Hub size checks and gated downloads) + optional **Firecrawl**
  - **ElevenLabs** (optional): enable + API key for cloud TTS when a character selects an ElevenLabs voice — see [ElevenLabs](../integrations/elevenlabs.md)
  - **TikTok**: client key/secret, redirect URI + OAuth port, publishing mode (inbox vs direct; inbox is supported for upload), optional **auto-upload after each render** — see [TikTok upload](../integrations/tiktok.md)
  - **YouTube**: separate enable; OAuth client ID/secret, redirect + port (default **8888**), default visibility, optional **#Shorts** tagging, optional **auto-upload after render** — see [YouTube upload](../integrations/youtube.md)
- **Model** (tab label; model downloads + dependencies)
  - Extra vertical spacing separates the **Download / Install / Clear** toolbar from the model stack, and **Model files location** from the combo grid ([`UI/tab_sections.py`](../../UI/tab_sections.py)).
  - **Local** vs **API** (top-right segmented toggle): **Local** keeps Hugging Face–backed combos, downloads, verify, and **Auto-fit**. **API** hides those local controls and shows the same **Generation APIs** panel as the **API** tab (keys + per-role provider/model, including the recommended **Gemini / SiliconFlow / Magic Hour / Inworld** options and env-var hints in tooltips) inside a **scroll area** so fields are not vertically crushed; the window also grows a bit in API mode. Persisted as `model_execution_mode` + `api_models` in `ui_settings.json` ([API generation](../integrations/api_generation.md)).
  - **Model files location** (below **Auto-fit**, local mode only): **Default** \| **External** — **Default** uses **`.Aquaduct_data/models`**; **External** shows a path, **Browse…**, **Apply** (saves to `ui_settings.json`), and **Detect** (list snapshots on disk). Title bar **Save** and **Run → Save settings** persist the same fields with everything else ([Config](../reference/config.md), [Models](../reference/models.md)).
  - **Script (LLM)** combo drives pipeline scripts, **🧠** field expansion, and **Characters → Generate with LLM** — the **current** selection is used even if you have not clicked **Save** yet ([`resolve_llm_model_id`](../../UI/brain_expand.py), [Models](../reference/models.md)).
  - **Image** = text-to-image (slideshow stills, SVD keyframes). **Video** = motion models (ZeroScope, Stable Video Diffusion). **Pro** mode (slideshow off) uses the **Video** slot for **multi-scene** output: **text-to-video** repos animate prompts directly; **img2vid** repos consume **keyframes** generated with the **Image** model. A legacy **slideshow + Pro** path can still use a T2I id from the Video slot for per-frame stills if both toggles are on in saved settings; **slideshow + Pro + SVD** uses the Image model for stills (the frame-stacking path does not run full img2vid).
  - **Auto-fit for this PC**: re-detects GPU/RAM and sets script, image, video, and voice dropdowns **and quant modes** using the same VRAM heuristics as the fit badges ([`src/models/hardware.py`](../../src/models/hardware.py) → `rank_models_for_auto_fit`), including **effective VRAM per kind** from the current **GPU policy** (Auto vs single) so it matches **My PC**. Skips grayed-out Hub entries; logs the chosen combo (with `quant=`) and **saves settings** (same as title bar **Save**). Appends a multi-line **[Aquaduct][inference_profile]** report ([`format_inference_profile_report`](../../src/models/inference_profiles.py)) so you can see per-role bands, profile ids, and resolved quant without starting a run — see [Inference profiles](../reference/inference_profiles.md) and [Quantization](../reference/quantization.md).
  - **Per-row quantization** (local mode): each Model row has **Automatic (fit this GPU)** plus a horizontal **slider** over enabled manual modes only (**low VRAM → higher quality**). Uncheck Automatic to choose a manual step; the label beside the slider shows the mode name (tooltip from the quantization policy). Kokoro-only voice rows keep **Automatic** locked and hide the slider. The **VRAM** label stays quant-aware (e.g. `~7-9 GB · NF4 4-bit`) with multiplier / fallback rationale in its tooltip. Values persist in `AppSettings.{script,image,video,voice}_quant_mode` ([Quantization](../reference/quantization.md)).
  - **Download ▾** menu: download the currently selected model by role (**script** / **image** / **video** / **voice**); **download all voice models** (curated TTS: Kokoro, MOSS-VoiceGenerator); **download all selected** (script + image + video + voice in one queue); **download all models** (full curated list); **verify checksums** for installed snapshots (selected or all under `models/`); plus **check Python dependencies** / **install dependencies** — **install** opens a **progress dialog** with live pip output, parsed “current package” hints (`pip_line_hint` in [`src/models/torch_install.py`](../../src/models/torch_install.py)), and a **progress bar** that stays **indeterminate** until pip prints a line with a **download percentage** (then shows **0–100%**). Same underlying install as `scripts/install_pytorch.py` + `requirements.txt` ([`UI/install_deps_dialog.py`](../../UI/install_deps_dialog.py)).
  - When the **Video (motion)** **Fit** badge is **VRAM Limit** (red) or **Risky** (amber), **Use lighter Motion / Video checkpoint** may appear — applies the curated smaller T2V id from **`next_smaller_repo_id`** ([`variant_fallback`](../../src/runtime/variant_fallback.py)).
  - Each model row shows a short **status badge**: **✓ On disk** (size-based), **✓ Verified** (after a good checksum run), or problem states such as **✗ Missing files** / **✗ Corrupt** / **⚠ Verify error**. Results persist in `data/model_integrity_status.json`. Verification also opens a **summary dialog** with expandable details ([models](../reference/models.md)).
  - Model dropdowns show local install status and Hub probe hints; downloads **skip** repos that already have a valid snapshot under `models/` and continue with the rest.
- **My PC**
  - Hardware summary: OS, CPU, RAM, and **GPU(s)** as plain lines (`Name (X.X GB)` per adapter from `list_cuda_gpus()`), not a detail table.
  - **GPU policy**: segmented **Auto** \| **Select GPU** ([`UI/gpu_policy_toggle.py`](../../UI/gpu_policy_toggle.py)) — **Auto** uses multi-GPU stage routing (script vs diffusion may use different devices; VRAM is not pooled). With **two or more** GPUs, **LLM** and **diffusion** are always on **different** ordinals. If the max-VRAM and compute heuristics would pick the **same** GPU, the **LLM** is placed on the **best other** CUDA device so a second GPU is not left nearly idle. Local **diffusion** **`auto`** offload ([`src/util/diffusion_placement.py`](../../src/util/diffusion_placement.py)) defaults to **sequential** CPU offload when multiple CUDA devices exist, to keep peak VRAM low on the diffusion GPU — unless **VRAM-first sharding** (below) steers **`auto`** toward full-GPU residency for peer submodule moves. **Select GPU** pins all local CUDA stages to the chosen device; the device combo is shown only in that mode when CUDA GPUs exist. Settings persist with the rest of the UI.
  - **VRAM-first sharding**: combo **Off** \| **VRAM-first multi-GPU** — maps to **`multi_gpu_shard_mode`** in `AppSettings` / `ui_settings.json`. When **VRAM-first multi-GPU**, **Auto** policy, and **≥ 2** CUDA devices apply, the pipeline may use experimental intra-model placement (Accelerate-balanced causal LM + diffusers text-encoder splits); expect slower runs and see [Hardware](../reference/hardware.md).
  - Per-model **Kind / Model / Speed / Fit / Why** uses `rate_model_fit_for_repo` + **effective VRAM** per kind (aligned with the **Model** tab). Deep policy notes: [`docs/hardware.md`](../reference/hardware.md).

## Settings persistence
Saved to:
- `ui_settings.json` (repo root; marked **hidden** on Windows and macOS after each save so it stays out of casual folder browsing)

## Theme
Applied via a global Qt stylesheet (QSS) built from [`UI/theme/palette.py`](../../UI/theme/palette.py) (`build_qss` / `resolve_palette`) with:
- near-black backgrounds
- default cyan accent `#25F4EE`
- default pink/red accent `#FE2C55`
- optional palette overrides configured in the Branding tab
- **`QTabWidget::tab-bar`** left offset so tab labels align with the content pane below (avoids visual overhang on the window edge)
- **`card`** surface for **`QFrame#SettingsSectionCard`** (see **Section composition** under Layout); inputs inside cards get a slightly lifted background so they stay distinct from the card fill. **`QLabel`** uses transparent background so nested labels do not paint solid slabs over cards.

**Outlined buttons (title bar + dialogs):** Small controls that would show **jagged** rounded rectangles under Fusion + QSS use **`TitleBarOutlineButton`** ([`UI/title_bar_outline_button.py`](../../UI/title_bar_outline_button.py)) — custom **`QPainter`** stroke with antialiasing. The factory **`styled_outline_button()`** applies palette colors from [`resolve_palette()` in `UI/theme.py`](../theme.py) for borderless dialogs and the Help tutorial (**Previous** / **Next** / **Close**). QSS sets `QPushButton[chrome="title_outline"]` to **no** stylesheet border so custom paint is not doubled up.

## Dialogs and alerts
- The main window is **frameless** with a custom title bar and **✕** (see [`UI/main_window.py`](../../UI/main_window.py)). **Modal alerts**, confirmations, the **Hugging Face token** prompt, **Preview** / **Storyboard Preview** windows, **Topics → Discover** approval dialogs, and the **model integrity** summary use the same pattern via [`UI/frameless_dialog.py`](../../UI/frameless_dialog.py): **`FramelessDialog`** (title + **`TitleBarOutlineButton`** ✕, drag the title bar to move) and helpers such as **`aquaduct_information`** / **`aquaduct_warning`** / **`aquaduct_question`**. Footer actions use **`styled_outline_button()`** for matching outlines. **`FramelessDialog._frameless_close_button`** exposes the ✕ when code must disable it (e.g. pip running). Styling includes `QDialog#FramelessDialogShell` in [`UI/theme.py`](../theme.py) (rounded card border).
- **Help / tutorial** ([`UI/tutorial_dialog.py`](../../UI/tutorial_dialog.py)): **Previous** / **Next** / **Close** use **`styled_outline_button()`**.
- **Download / import progress** ([`UI/download_popup.py`](../../UI/download_popup.py)): **Pause** and **✕** use **`styled_outline_button()`**.
- **Install dependencies** ([`UI/install_deps_dialog.py`](../../UI/install_deps_dialog.py)): footer **Close** uses the same chrome.
- **System file/folder pickers** (`QFileDialog`) remain the OS native dialogs.
